# Security Policy

## Supported versions

Security fixes are applied to the current main branch and tagged release lines.

## Reporting a vulnerability

Do not open a public issue for a security vulnerability.

Report the problem through the repository's private security workflow if available, or contact the maintainers through the project's established private channel.

Include:

- A short description of the issue
- Affected component or file path
- Reproduction steps
- Any relevant logs or screenshots, with secrets removed

## What to avoid

- Do not include API keys, tokens, passwords, database URLs, or lease tokens in reports
- Do not share full crash dumps if they contain secrets or local paths
